sparQuery

Posted : 2005-05-18 00:00 / Category : [開発日誌]
ここ1ヶ月くらいボチボチ取り組んでいるsparQuery という個人プロジェクトについて。
ソーシャルクエリシステム(SQS)。
以下に、ToDo を記録。
3番目までを、5月中に実装して、 β 公開して、軽くプロモーションする予定。
4番目は、内部仕様なので、追々。でも、重要。
書いてないけど UI (ヴィジュアル含む)の最適化も重要。

ToDo(俺が欲しい順)

  1. クエリ管理
    XMLHttpRequest で実装予定。
    クエリにコメントなど付けられるようにしたい。
  2. RSS
    現在、location.hash でブックマークできるが、
    やはり RSS も欲しい。俺が。
  3. クエリソーシャライザ
    クエリのソーシャリティを分析し、
    ソーシャライゼイションをアシストする者。
    例えば、クエリ作成中に補完候補を出す。
    現在、Google suggest のみ実装済だが、
    本人、他人、のクエリデータベースからも補完候補を出したい。
    また、結果一覧のほうにも、ソーシャリティを表示する予定。
    それで初めて、ソーシャルになる。
  4. 検索Proxyの改善
    現在、SOAP で GoogleAPI とやり取りして、検索結果を取得している。
    GoogleAPI には制限があるし、SOAP にメリットを感じないので、
    専用の検索 Proxy (チカラ技)を構想中。
    誰か既に作ってるんじゃないかと期待。

ツッコミ

1: master (06/01 10:54)
予定はズレズレ。1,2,3 をすっ飛ばして、4. を opensearch で実装。
2: master (06/23 23:50)
進捗率:
1(70%), 2(20%), 3(0%), 4(90%).
今後の予定:
まず、3. に着手。
次に、2. に着手。

完了したら、新たに以下のタスクを加える。
5. スコアリング
検索結果の各レコードに、サーバ側でスコアを付加してクライアントに返す枠組。
スコアリングは、Bayesian にする。
